--- --help ---
Extract plain text from docx, odt, epub, xlsx, pptx, rtf, html, and ipynb files.

Usage: extract-text [OPTIONS] <PATH>

Arguments:
  <PATH>
          The path to the input file

Options:
      --format <FORMAT>
          Override the format instead of inferring from the file extension
          
          [possible values: docx, odt, epub, xlsx, pptx, rtf, html, htm, ipynb]

      --service-limits
          Apply the file-parser HTTP service's decompression and output caps.
          
          The zip-format extractors (docx, odt, epub, xlsx, pptx) can enforce a per-entry decompressed-size cap, a compression-ratio bomb threshold, and a total-extracted-text cap as a defense against hostile inputs. The service enables all three; this binary enforces only a 1 GiB per-entry cap by default so that large legitimate inputs are not refused. This flag restores the service's tighter caps, which is useful when reproducing a service-side rejection locally. Wins over --max-entry-bytes when both are given.

      --max-entry-bytes <BYTES>
          Override the per-zip-entry decompressed-size cap (default 1 GiB). The total-output cap and the compression-ratio check stay disabled, so this bounds memory per zip entry only. Ignored for non-zip formats (rtf, html, ipynb) and when --service-limits is set

  -h, --help
          Print help (see a summary with '-h')

  -V, --version
          Print version
